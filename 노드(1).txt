※노드는 싱글스레드이다.그럼 백그라운드에서 실행되는 함수는 어떤 스레드에서 실행되는가
1) node는 주요 작업을 처리하는 주 스레드(메인 스레드)가 단 하나만 존재하는 싱글 스레드 환경이다.
즉 js 코드 자체는 하나의 스레드에서 실행되며, 동시성을 처리하는 방식은 다른 메커니즘을 사용함.
2)백그라운드에서 비동기 작업을 처리하는 방식으로 libuv 라이브러리를 사용함. 비동기 작업이 실행되면 이벤트 루프가 메인스레드와는 별개로 작업을 처리하는 
별도의 큐에 전달하여 처리한다.
3)작업이 끝나면 콜백함수를 이벤트 큐에 넣고 메인스레드가 이를 처리한다.

※노드를 사용할때 브라우저에서 사용하는 js와 서버에서 사용하는 js의 차이점
1) 기본적인 문법(변수선언,함수,조건문..)은 동일하지만, 실행 환경과 사용 가능한 API가 다르기 때문에 차이점이 있다.
2)html 내부의 js -> 주로 브라우저에서 실행되 DOM,window,document 객체와 같은 웹 API를 사용할 수 있다.
2-1)node.js에서 실행되는 js->위와 같은 객체가 없고, [fs,http,net,events]등 서버와 관련된 모듈을 제공하고 require를 사용해 외부 모듈을 가져와 작업을 수행
3)브라우저와 서버간의 상호작용
3-1)html ->사용자와의 상호작용(버튼 클릭)을 처리함. 서버와의 통신은 보통 ajax,fetch api를 사용해 이뤄짐
3-2)node ->서버측에서 http요청을 수신하고 응답을 보낸다. -> 클라이언트와의 http 통신을 처리하거나, 데이터베이스와 상호작용하여 동적 웹 페이지를 생성하는 작업
