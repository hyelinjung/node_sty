->create schema [데이터베이스명] mysql에서는 데이터베이스와 schema는 같은 개념
->int 정수를 말함, 소수까지 저장하고 싶으면 float나 double 사용
->varchar 은 가변길이 varchar(10)은 0~10 길이의 문자열을 넣을 수 있음, char은 고정길이.. 주어긴 길이보다 짧은 문자열을 넣으면 부족한 만큼 스페이스가 채워짐
->text은 긴 글을 저장할 때 사용됨...varchar와 헷갈리는데,, 몇백 자 이내의 문자열은 보통 varchar로 많이 처리하고 더 길면 text사용
->unsigned 옵션으로 숫자는 기본적으로 음수 범위를 지원하는데 양수만 적용하겠다는 뜻
->unique index 해당 값이 고유해야 하는 지에 대한 옵션..PK도 고유해야 하지만 pk는 자동으로 이 옵션을 포함해 따로 적지 않는다.
->다른 테이블의 기본 키를 저장하는 컬럼을 외래 키라고 함 ->CONSTRAINT 제약 조건명 FOREINGN KEY [컬럼명] REFERENCES [참고하는 컬럼명]

일대일 -> 사용자 한 명은 자신의 정보를 담고 있는 테이블과 관계과 있고, 정보 테이블도 한 사람만 가리킴
다대다 -> 한 계시글에는 해시태그가 여러개 달릴 수 있고, 한 해시태그도 여러 게시글에 달릴 수 있다.

※(1)module.exports로 내보낼 값을 보내면 해당 파일을 리턴한 후 내보낼 값만 사용할 수 있는지 VS 파일이 아니라 내보낼값 자체를 반환하는지
->module.exports는 해당 모듈에서 외부로 내보낼 값을 정의하는 것으로 이 값은 함수, 객체 ,값 등 어떤 것이든 될수 있다. require은 그 값 자체를 반환
즉 함수, 객체, 값을 가져온 후 이를 사용함
const User = require('./models/userModel')(sequelize, DataTypes); 
이 코드는 아래 코드로 수정가능
let UserModel = require('./models/userModel');
let User = UserModel(sequelize, DataTypes);
